# 📥 收件箱 (Inbox) 说明

这里不是普通笔记的默认落点，而是自动导入与待分拣内容的暂存区。

## 主要作用

这里主要承接 4 类内容：

1. 剪藏、邮件、同步脚本落地的外部材料
2. AI 生成但尚未定类的临时产物
3. 需要稍后判断去向的零散资料
4. 少量待分拣附件或中间文件

## 工作流程

### 整理原则

普通手写笔记不再默认进入这里：

- 当天记录、闪念、临时草稿 -> `Calendar/Daily notes/`
- 需要跟踪的事项 -> `Calendar/Tasks/`
- 已知去向的内容 -> 直接在目标文件夹创建

Inbox 只做短暂停靠，定期清空并分流：

- 外部材料、剪藏 -> [[Sources]]
- 项目推进资料 -> [[Spaces]]
- 可复用的长期沉淀 -> [[Cards Readme]]
- 导航、索引、总览 -> [[Atlas]]

### 心理效应

相比于cards文件夹，inbox文件夹更多地具备action属性，以及解压作用。就算只是简单地将inbox挪到cards，但这种清空inbox带来的心理解压的感觉，是很不错的。

**目标：保持 Inbox 定期清空 (Inbox Zero)，单条内容最长停留不超过 7 天。**
