> `Atlas/MOCs/` 是本库的 MOC 入口层。这里主要解决“从哪里开始逛”“主题总入口放哪里”“哪些 MOC 该长期维护”这 3 个问题。

## 先看哪几页

- [[∑ 全库MOC索引]]：全库导航页，按主题找入口
- [[MOC]]：MOC 方法论说明，解释它是什么、为什么存在
- [[🍀 花园导览]]：面向读者的外部逛园入口

## 这个文件夹放什么

这里优先放两类 MOC：

- 跨目录、跨主题的总入口 MOC
- 已经足够稳定、值得长期维护的主题 MOC

如果某个 MOC 只服务于单一项目、单一 Area、单一 Resource 子目录，优先放在原目录，不强行上收到 `Atlas/MOCs/`。

## 当前的 MOC 分层

大致分 3 层：

- 根入口：[[∑ 全库MOC索引]]
- Atlas 主题入口：如 [[∑ 技术栈全景图]]、[[∑ 效率工具生态]]、[[∑ 资源与阅读索引]]
- 各目录内部入口：分散在 `Spaces/`、`Cards/`、`Sources/`、`Calendar/` 的 `∑` 页或索引页

## 命名建议

- `∑ 主题名`：正式主题 MOC，长期维护
- `moc - xxx`：实验性、临时性或工作流型索引
- `xxx 索引` / `xxx 地图`：标题更自然时可用，但尽量不要和现有 `∑` 页重复造轮子

## 维护原则

- 一个主题尽量只有一个主入口，避免 `MOC / 索引 / 地图` 并行重复
- MOC 以导航为主，不把自己写成正文堆场
- 先挂稳定专题，再挂代表性笔记，不追求把所有相关链接都塞进去
- 新主题先补到现有 MOC；只有在跨目录、跨主题且会长期扩展时，再新建独立 MOC
