# 📂 文件夹结构总览

本文档提供了整个知识库的文件夹结构说明，每个文件夹都有详细的用途介绍和使用指南。

## 🗂️ 核心文件夹

### [[_ Atlas Readme|🗺️ Atlas - 知识图谱]]

存放导航、MOC、Canvas、Base、分类索引等全库级骨架内容。这里负责“指路”，不再承接普通正文型工作笔记。

### [[_ Cards Readme|🃏 Cards - 卡片笔记]]

存放原子化的知识卡片和零散思考，是自下而上笔记结构的核心池子。通过标签、链接等方式组织，等待时机成熟后结构化整理。

### [[_ Sources Readme|📚 Sources - 来源文献]]

存放文献笔记和外部资源的引用摘要。按资源类型分类，包含书籍、文章、课程、剪藏等各类来源材料的处理和思考。

### [[_ Spaces Readme|🗂️ Spaces - PARA空间]]

基于PARA方法论组织的行动导向空间，包含项目(Project)、领域(Area)、资源(Resource)、归档(Archive)四个层次的内容管理。

## 📅 时间管理

### [[_ Calendar Readme|📅 Calendar - 日历管理]]

存放时间流相关内容，是默认手写入口。包含 `Daily notes`、`Tasks`、`Plan & Review` 等时间序列记录。
其中 `Calendar/Tasks/Archive/Duplicates/` 只用于收口自动生成或历史遗留的重复任务副本，不作为新的任务入口。

## 🛠️ 辅助功能

### [[_ Extras Readme|⚙️ Extras - 附加资源]]

存放支持知识库运行的各种附加资源，包括模板、脚本、提示词、附件等技术支持文件。

### [[_ 📥 Inbox Readme|📥 Inbox - 收件箱]]

只保留为自动导入和待分拣暂存区。普通笔记不再默认创建到这里，目标是持续清空。

## 🌸 导览指南

### [[_ 🍀 花园导览 Readme|🍀 花园导览 - Garden Guide]]

数字花园的入口和导航中心，提供主题导览和使用指南，帮助访问者更好地漫游和探索知识库。

## 📊 整体架构

本知识库采用 [[ACCESS 笔记组织法]]，结合以下方法论：

- **PARA方法**: 按行动性组织信息（Project/Area/Resource/Archive）
- **Zettelkasten**: 原子化卡片笔记和双向链接
- **Digital Garden**: 渐进式生长的开放知识空间
- **Building a Second Brain**: CODE方法（Capture/Organize/Distill/Express）

## 🔄 信息流动

```
📅 Calendar/Daily notes → Calendar/Tasks / 🃏 Cards / 🗂️ Spaces
📥 Inbox → 📚 Sources / 🃏 Cards / 🗂️ Spaces
🗺️ Atlas → 负责导航，🍀 花园导览 → 负责对外入口
```

## 💡 使用建议

1. **新手入门**: 从 [[🍀 花园导览]] 开始，了解整体架构
2. **日常使用**: 新想法先写进 `Calendar/Daily notes/`，需要持续跟踪时再转任务或项目
3. **收件箱处理**: [[📥 Inbox]] 只接自动导入和待分拣内容，不手工堆普通笔记
4. **知识建构**: 在 [[🃏 Cards]] 中培育想法，成熟后再通过 [[🗺️ Atlas]] 建立导航
5. **项目管理**: 用 [[🗂️ Spaces]] 的 PARA 结构管理具体行动
6. **整理入口**: 不确定去哪时，先看 [[这条笔记该去哪]]
7. **重复任务**: 看到 `Archive/Duplicates` 时把它当作审计区，不再继续往里面新建内容

## 根目录规则

- 根目录只保留仓库级文件、许可证和配置文件
- 普通笔记、任务总结、提示词、项目文档一律不再放根目录
