# 📂 文件夹结构总览

本文档是本库当前的目录路由图：它回答“不同类型内容的 canonical home 在哪里”，并把受保护的运行区、同步区、任务区和私人区从“普通清理对象”里明确排除。

## 一句话

根目录只放公开入口、仓库规则和运行配置；普通内容进入 `Calendar/`、`Sources/`、`Cards/`、`Spaces/` 或 `Atlas/`，支撑资产进入 `Extras/`，长期 agent / 系统控制面进入 `_system/`。

## 根目录：入口和配置，不放普通笔记

| 路由 | canonical home | 用途 |
|---|---|---|
| 对外首页 | `README.md`、`AboutTheGarden.md`、`🍀 花园导览/` | 公开读者入口、花园导览和本库教程 |
| 协作规则 | `AGENTS.md`、`GEMINI.md -> AGENTS.md`（仅需 Gemini CLI 兼容时存在） | agent / CLI 进入仓库后的全局规则 |
| 命令入口 | `justfile` | 日常维护命令真源，先用 `just --list` 查看 |
| 许可证 | `LICENSE` | 开源许可文本 |
| Git / LFS / 类型配置 | `.gitignore`、`.gitattributes`、`_types/`、`mdbase.yaml` | 仓库、附件、Base / mdbase 相关配置 |
| 运行配置 | `.obsidian/`、`.agents/`、`skills-lock.json` | Obsidian / agent / skill 运行入口；不是普通内容区；`.claude/`、`.claudian/` 若出现按同类运行配置处理 |
| 本地忽略输出 | `.tmp/`、`.DS_Store` 等 | ignored 本地缓存 / 报告，不作为内容路由目标 |

根目录普通 Markdown 不再是合法落点；最近一次根目录路由已经把日报总结、书摘、AI Agent 说明、系统清理报告等移回对应主干。

## 内容路由表

| 内容类型 | canonical home | 说明 |
|---|---|---|
| 当天想到的事、流水、手写草稿 | `Calendar/Daily notes/` | 默认手写入口；按日期稳定存放，不跨历史日记批量重写 |
| 周期计划、复盘、AI summaries | `Calendar/Plan & Review/` | 日 / 周 / 月 / 年复盘和自动生成总结；AI summaries 进入 `Calendar/Plan & Review/AI summaries/<year>/` |
| 任务、TaskNotes、Linear 镜像 | `Calendar/Tasks/` | 任务真相源；重复副本只进入 `Calendar/Tasks/Archive/Duplicates/` 审计区 |
| TaskNotes 视图和工作台 | `Atlas/Bases/TaskNotes/` | 任务结果集和视图层，不直接替代 `Calendar/Tasks/` 源笔记 |
| 外部文章、书、课程、剪藏 | `Sources/` | 来源层，保留材料 provenance；来源消化后再进入 LLM Wiki / Cards / Spaces |
| Web Clipper 快速导入 | `Sources/Clippings/` | append-only 导入池；不要为清空 backlog 机械移动或删除 |
| 同步导入原文 | `Sources/CuboxSync/`、`Sources/GithubStarsSync/`、`Sources/*Sync/` | 同步 / 导入来源默认保留原路径，不批量改名或合并 |
| AI / 技术来源和样本 | `Sources/AI/`、`Sources/DataSamples/`、`Sources/Skills/` | 来源、数据样例和 skill 知识卡；`Sources/Skills/` 不等同于可运行 skill |
| 原子化常青卡片 | `Cards/` | 扁平常青笔记池；成熟主题由 Atlas / Spaces 承接导航，不在 Cards 下硬分深层目录 |
| 正在推进的项目 | `Spaces/1-Project/` | 项目资料、运行证据、交付上下文 |
| 长期负责领域 | `Spaces/2-Area/` | 数字花园、健康、云服务等持续维护主题 |
| 资源库 | `Spaces/3-Resource/` | 主题资源和长期参考材料 |
| 历史归档 | `Spaces/4-Archive/` | 历史项目和旧资料；不按年龄自动删除 |
| 公开发布内容 | `Spaces/Published/` | 已发布文章、草稿和发布工作流相关内容 |
| 私人内容 | 私密空间 | 隐私边界；不进入公开导览细节，也不作为通用清理目标 |

## Atlas：导航、编译层和可视化工作台

| 路由 | canonical home | 说明 |
|---|---|---|
| 全库 MOC 和主题索引 | `Atlas/MOCs/` | 内容地图、主题枢纽和导航页 |
| 分类 / fileclass | `Atlas/Categories/` | 分类控制面和分类 Base |
| LLM Wiki 编译层 | `Atlas/LLM Wiki/` | source summary、concept、query、synthesis、reading map 和 promotion 队列 |
| Base 工作台 | `Atlas/Bases/` | `.base` 视图层；路径必须是相对路径并保留扩展名 |
| Canvas 工作流 | `Atlas/Canvas/` | Obsidian Canvas；`.canvas` 必须保持严格 JSON，file nodes 不能新增缺失目标 |
| 绘图资产 | `Atlas/Draws/` | Canvas / Excalidraw / 导出图 / 绘图说明等可视化素材 |

`Atlas/LLM Wiki/`、`Atlas/Bases/`、`Atlas/Canvas/` 和 `Atlas/Draws/` 都是路径敏感区：可被索引、审计、补链接，但不能被当成普通目录减法目标。

## Extras：脚本、模板、附件和运行资产

| 路由 | canonical home | 说明 |
|---|---|---|
| 维护脚本 | `Extras/Scripts/` | 校验、发布、整理和报告脚本；优先 safe / dry-run，再 mutating |
| Obsidian 模板 | `Extras/Templates/` | Daily notes、Templater、QuickAdd、Buttons、Dataview / Base 或脚本引用的模板；不要因为空或旧就删 |
| 附件和静态资产 | `Extras/Attachments/` | 图片、图标、配置、导入附件等；大文件遵守 Git LFS |
| 跨设备运行资产桥 | `Extras/DeviceAssets/` | Obsidian Sync 不稳定同步的 `.agents/`、Launcher Skills、agent config 等源包；真实应用走 `just device-assets-status` / dry-run / apply |
| 插件 / Excalidraw / Prompt / 小数据 | `Extras/PluginAssets/`、`Extras/Excalidraw/`、`Extras/Prompts/`、`Extras/Data/` | 插件补全、绘图、Prompt 和脚本数据支持 |

`Extras/DeviceAssets/` 只放可同步的源资产，不放 token、workspace 状态、cache 或单设备 runtime。`Extras/Scripts/` 和 `Extras/Templates/` 是运行资产，不是“杂物清理区”。

## \_system 与 agent 层

| 路由 | canonical home | 说明 |
|---|---|---|
| 稳定 agent 上下文 | `_system/meta/` | [[_system/meta/AGENT_QUICKSTART\|AGENT_QUICKSTART]]、[[_system/meta/CURRENT_FOCUS\|CURRENT_FOCUS]]、[[_system/meta/MEMORY\|MEMORY]]、[[_system/meta/TOOLS\|TOOLS]] 等 |
| 过程记忆 | `_system/memory/YYYY-MM-DD.md` | 当天上下文、临时判断和任务接力 |
| 计划、worklog、结构说明 | `_system/docs/` | roadmap、Goal Maker、结构治理、系统审计和长期控制面 |
| 多 bot 专区 | `_system/agents/` | Hermes bot 的 README / MEMORY / CONFIG / TASK\_LIFECYCLE / STATE |
| 可运行 repo-local skills | `.agents/skills/` | 真实可执行 skill；与 `skills-lock.json` 对齐 |

## 信息流动

```text
Calendar/Daily notes
  → Calendar/Tasks / Cards / Spaces

Sources / Sources/Clippings / Sources/*Sync
  → Atlas/LLM Wiki
  → Cards / Spaces / public guide when mature

Cards / Spaces
  → Atlas/MOCs / Atlas/Bases / 🍀 花园导览

Scripts / Templates / DeviceAssets
  → Extras
  → justfile / AGENTS / _system/meta/TOOLS for operator discovery
```

## 保护区不是通用清理目标

以下区域可以做只读审计或按明确 manifest 小批处理，但不能因为“目录多 / 看起来旧 / 低价值”就自动移动、删除或合并：

- `.obsidian/`、`.agents/`
- `Extras/Templates/`、`Extras/DeviceAssets/`、`Extras/Scripts/`
- `Atlas/Bases/`、`Atlas/Canvas/`、`Atlas/Draws/`、`Atlas/LLM Wiki/`
- `Calendar/Daily notes/`、`Calendar/Tasks/`
- `Sources/Clippings/`、`Sources/*Sync/`
- 私密空间目录
- Git LFS 管理的大附件、冲突副本、入链 redirect stub

## 使用建议

1. 新手从 [[🍀 花园导览]] 和本页开始，理解顶层路由。
2. 不确定一条新笔记去哪时，先看 [[这条笔记该去哪]]。
3. 手写内容先进入 `Calendar/Daily notes/`；外部材料先进入 `Sources/` 或 `Sources/Clippings/`。
4. 稳定判断进入 `Cards/`、`Spaces/` 或 `Atlas/LLM Wiki/`；成熟后再回桥到 MOC、Base 或公开导览。
5. 大规模整理先出 dry-run / manifest / reference scan，再执行小批移动。

## 入口对齐

- 对外叙事以 `README.md` 和 [[🍀 花园导览]] 为入口；本页只解释目录路由，不替代首页。
- agent / operator 边界以 `AGENTS.md`、[[_system/meta/AGENT_QUICKSTART|AGENT_QUICKSTART]]、[[_system/meta/TOOLS|TOOLS]] 为准；命令真源是 `justfile` / `just --list`，本页只描述目录路由。
- M3 后的结构批次是 no-op 收口；M6 已收敛 operator entrypoint，README 公共导航最终措辞由后续 README/workbench finalization 单独处理。
