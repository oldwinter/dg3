# 这条笔记该去哪

当你准备新建一条内容时，先判断它属于哪一种，再决定落点。

## 一句话规则

- 一种内容只保留一个主入口
- 根目录不放普通笔记
- 任务只认 `Calendar/Tasks`

## 快速分流

| 内容类型 | 放哪里 | 说明 |
|---|---|---|
| 当天想到的事、闪念、临时草稿 | `Calendar/Daily notes/` | 默认手写入口，先记录再整理 |
| 需要持续跟踪的事项 | `Calendar/Tasks/` | 本地任务、Linear 镜像、长期待办都收口到这里 |
| 外部文章、书、课程、剪藏 | `Sources/` | 外部材料及其消化结果 |
| 自动导入、待分拣内容 | `📥 Inbox/` | 只做暂存，不做长期存放 |
| 可复用的概念、结论、术语、方法 | `Cards/` | 永久笔记，保持扁平 |
| 正在推进的项目或长期负责领域 | `Spaces/` | Project / Area / Resource / Archive |
| 导航页、MOC、Canvas、Base、分类索引 | `Atlas/` | 负责全库导航，不承接普通正文 |
| 模板、附件、脚本、提示词 | `Extras/` | 支撑材料 |
| 面向读者的导览与公开入口 | `🍀 花园导览/` | 对外展示，不混个人工作残留 |

补充：
`Calendar/Tasks/Archive/Duplicates/` 只收历史重复副本，不手工创建，也不视为任务真相源。

## 不要放在根目录的内容

- 今日/周任务总结
- 提示词和操作手册
- 项目文档与架构图
- 临时工作笔记
- 外部资料和剪藏

## 两个常见判断

1. 还没想清楚，但它是外部材料或自动化落地结果：先去 `📥 Inbox/`
2. 还没想清楚，但它是你今天手写下来的：先去 `Calendar/Daily notes/`
